## Executive Summary

**Issues found:** 1 High, 4 Medium-high, 6 Medium, 5 Low.

**Top 3 to address:**

1. **Missing `:focus-visible` styles** across all interactive elements ‚Äî keyboard users have no visible focus indicator (High).
2. **Image weight ~30MB total** ‚Äî `spark-broadside.png` is 7MB, several JPGs over 1MB, no lazy loading (Medium-high).
3. **Mobile nav checkbox hack with `aria-hidden`** ‚Äî screen reader users on mobile can't tell if nav is open (Medium).

**Overall quality:** strong design system, weak accessibility wiring. The aesthetic is genuinely distinctive; the wiring around it has the kind of gaps you'd find in a portfolio site iterated for visual impact more than for AT users.

**Recommended next steps:** run `/harden` for focus + ARIA + edge cases, then `/optimize` for image compression and lazy loading.

## Findings by Severity

### High

#### A11y-1 ¬∑ Missing `:focus-visible` styles
- **Location:** `styles.css` ‚Äî entire stylesheet has no `:focus-visible` rule
- **Category:** Accessibility / WCAG 2.4.7
- **Description:** No explicit focus indicators on `a`, `.contact-link`, `.callout`, `.location-pill`, `.nav-toggle-button`, or any other interactive element. The brutalist design and `a { color: inherit; text-decoration: none; }` mean keyboard users see no focus.
- **Impact:** Keyboard navigation invisible. Fails WCAG AA 2.4.7 (Focus Visible).
- **Fix:** Add a global `:focus-visible` rule using `outline: 2px solid var(--brass); outline-offset: 2px;` plus targeted overrides on the amber banner sections (use ink-deep there, since brass disappears on amber).
- **Command:** `/harden`

#### Perf-1 ¬∑ Large unoptimised images
- **Location:** `img/spark-broadside.png` (7.0 MB), `img/vox-arboris-clip.mp4` (12 MB), `img/mayor-manor.jpg` (2.3 MB), `img/tardis-closeup.jpg` (1.4 MB), `img/spif-performing.jpg` (1.1 MB), `img/cubic-reset-*.jpg` (~1 MB each)
- **Category:** Performance
- **Description:** Total media payload is ~30 MB across ~25 assets. The 7 MB PNG is the worst offender ‚Äî at 2048√ó2048, almost certainly a linocut illustration that compresses dramatically as WebP at quality 80.
- **Impact:** Slow LCP on mobile/slow connections. Multi-second waits on flaky cell connections.
- **Fix:** Convert PNG to WebP (`cwebp -q 80`), recompress JPGs above 1 MB, optionally provide `<picture>` with WebP source + JPEG fallback. Target: site under 5 MB total.
- **Command:** `/optimize`

### Medium-High

#### A11y-6 ¬∑ Video lacks captions
- **Location:** `index.html:295-304` ‚Äî `<video controls preload="metadata">` with no `<track>`
- **Category:** Accessibility / WCAG 1.2.2
- **Description:** The Vox Arboris clip plays without captions or transcript.
- **Impact:** Deaf/HoH users excluded from the video content. Required for prerecorded synchronised media.
- **Fix:** Add `<track kind="captions" src="vox-arboris-clip.vtt" srclang="en" label="English">`. Even an empty/placeholder VTT shows intent; ideally write actual captions or note "instrumental music, no dialogue" if applicable.
- **Command:** `/harden`

#### Perf-2 ¬∑ Images lack `loading="lazy"`
- **Location:** All `<img>` tags in `index.html`
- **Category:** Performance
- **Description:** All ~25 images load eagerly on initial page load even though most are far below the fold.
- **Impact:** Wasted bandwidth, slower LCP, slower interaction-ready time.
- **Fix:** Add `loading="lazy"` to every image except the hero (`/img/hero-poster.jpg`); add `loading="eager" fetchpriority="high"` to the hero.
- **Command:** `/optimize`

#### Perf-3 ¬∑ Images lack explicit `width`/`height` attributes
- **Location:** All `<img>` tags in `index.html`
- **Category:** Performance / CLS
- **Description:** CSS `aspect-ratio` + `position: absolute; inset: 0` on `.plate img` enforces dimensions in practice, but explicit `width`/`height` attributes are still best practice for CLS resilience if CSS load is delayed.
- **Impact:** Edge-case layout shift; minor.
- **Fix:** Add `width` and `height` attributes matching the image's natural pixel dimensions to every `<img>`.
- **Command:** `/optimize`

#### Perf-4 ¬∑ Google Fonts loaded render-blocking
- **Location:** `index.html:28`
- **Category:** Performance
- **Description:** `<link href="https://fonts.googleapis.com/...">` blocks first paint until the fonts CSS round-trips.
- **Impact:** Slower First Contentful Paint, especially on flaky connections.
- **Fix:** Either self-host the fonts with `@font-face` + `font-display: swap` (also removes Google as a deploy dependency), or `<link rel="preload" as="style">` the fonts CSS.
- **Command:** `/optimize`

### Medium

#### A11y-2 ¬∑ No skip-to-content link
- **Location:** `index.html:33`
- **Category:** Accessibility / WCAG 2.4.1
- **Description:** Keyboard users must tab through the entire sidebar nav before reaching main content on every page load.
- **Impact:** Friction for keyboard and screen reader users.
- **Fix:** Add `<a href="#main" class="skip-link">Skip to content</a>` at the top of `<body>`; style it visually hidden but focus-visible.
- **Command:** `/harden`

#### A11y-3 ¬∑ Mobile nav checkbox hack with `aria-hidden`
- **Location:** `index.html:34-38`
- **Category:** Accessibility
- **Description:** The `<input type="checkbox" aria-hidden="true">` + `<label>` pattern for the mobile nav toggle is hidden from screen readers. The label has `aria-label="Open index of works"` but no `aria-expanded` state.
- **Impact:** Mobile screen reader users can't tell if the nav is open or closed.
- **Fix:** Replace with `<button aria-expanded="false" aria-controls="sidenav">Index</button>` + 5 lines of JS to toggle `aria-expanded` and a CSS class on `<body>`. Modern, accessible, and barely more code.
- **Command:** `/harden`

#### A11y-1b ¬∑ Sidebar group headings `aria-hidden`
- **Location:** `index.html:46, 50, 54` ‚Äî `<li class="sidenav-group" aria-hidden="true">Upcoming|Current|Retired</li>`
- **Category:** Accessibility
- **Description:** The "Upcoming / Current / Retired" tense groupings are hidden from AT entirely. Sighted users get meaningful structure; AT users get a flat list.
- **Impact:** Loss of meaningful organization for screen reader users.
- **Fix:** Use `<li role="separator" aria-label="Upcoming">` (announces the separator) or restructure as semantically grouped `<ul>` with `<h3>` group titles inside a wrapping `<section>`.
- **Command:** `/harden`

#### A11y-8 ¬∑ Contrast ‚Äî `.hero-meta-secondary` at 0.7 opacity on amber
- **Location:** `styles.css:480` ‚Äî `.hero-meta-secondary { opacity: 0.7; font-size: 0.66rem; }`
- **Category:** Accessibility / WCAG 1.4.3
- **Description:** Dark navy text (`var(--ink-deep)`, oklch 0.12) at 0.7 opacity on amber background (oklch 0.68 0.15 50). Likely just under 4.5:1 for small text.
- **Impact:** May fail WCAG AA for small text.
- **Fix:** Verify with a contrast checker. If below 4.5:1, drop the opacity or boost it to 0.85.
- **Command:** `/harden`

#### A11y-9 ¬∑ Contrast ‚Äî `.plate .caption` on dark overlay
- **Location:** `styles.css:666` ‚Äî `color: var(--paper-faint)` (oklch 0.55) on overlay `oklch(0.10 0.025 255 / 0.85)`
- **Category:** Accessibility / WCAG 1.4.3
- **Description:** Captions are intentionally muted but borderline at 0.65rem font size.
- **Impact:** Could fail or be marginal for AA.
- **Fix:** Verify; bump caption color to `var(--paper-dim)` (oklch 0.78) if marginal.
- **Command:** `/harden`

#### Theme-3 ¬∑ Sidebar uses hardcoded colors instead of tokens
- **Location:** `styles.css:1066, 1067, 1077, 1098, 1119, 1130, 1134-1135, 1173-1174` ‚Äî `oklch(0.91 0.19 95)` (yellow) + `oklch(0.32 0.16 145)` (green) repeated ~10 times
- **Category:** Theming
- **Description:** The sidebar's yellow + green never get extracted to design tokens, so they can't participate in `data-mood` swaps and are tedious to update.
- **Impact:** Maintenance friction; theming inconsistency.
- **Fix:** Extract to `--sidebar-bg`, `--sidebar-ink`, `--sidebar-ink-soft` at `:root`. Optional: override per mood.
- **Command:** `/normalize`

### Low

#### A11y-10 ¬∑ Touch targets in mobile nav are ~22-32px high
- **Location:** `styles.css:1212-1215` ‚Äî `.sidenav-list a { padding: 0.6rem 0; font-size: 0.7rem; }` in mobile breakpoint
- **Category:** Accessibility / WCAG 2.5.5
- **Description:** Padding + font-size combine to roughly 30-32px tall, just under the 44√ó44px recommended target.
- **Impact:** Mistapped links on touch screens.
- **Fix:** Increase padding to 0.85rem or set `min-height: 44px`.
- **Command:** `/harden`

#### Theme-4 ¬∑ `data-mood="dusk"` body background is implicit
- **Location:** `styles.css:116-128, 217` ‚Äî velvet/playa/foundry set body background explicitly; dusk inherits the unscoped base rule
- **Category:** Theming
- **Description:** Asymmetric: three of four moods have explicit body backgrounds; the default doesn't. If load order shifted, dusk could break silently.
- **Impact:** Maintenance fragility.
- **Fix:** Add `html[data-mood="dusk"] body { background: ... }` or move the base body gradient into each mode rule.
- **Command:** `/normalize`

#### Perf-7 ¬∑ CSS unminified
- **Location:** `styles.css` (1252 lines, ~35 KB)
- **Category:** Performance
- **Description:** Cloudflare may auto-minify on serve, but explicit minification + content hash would improve cacheability.
- **Impact:** Marginal ‚Äî 5-10 KB on the wire.
- **Fix:** Optional; not a real bottleneck.

#### AP-2 ¬∑ `.status-pill` is the most generic element
- **Location:** `styles.css:790-811` ‚Äî `border-radius: 999px`, animated colored dot prefix
- **Category:** Anti-pattern (mild)
- **Description:** The rounded pill with colored dot is contemporary but somewhat templated ‚Äî the only element that breaks the brutalist hard-edge vocabulary.
- **Impact:** Minor visual inconsistency.
- **Fix:** Optional. Replace `border-radius: 999px` with `border-radius: 0` to match the rest of the brutalist vocabulary; or rename to `status-badge` to differentiate from `location-pill`.
- **Command:** `/normalize`

#### AP-3 ¬∑ Two "pill" components serve different roles
- **Location:** `.location-pill` (hard-edged dark stamp) vs `.status-pill` (rounded amber badge)
- **Category:** Anti-pattern (mild)
- **Description:** Both called "pill" but visually distinct. Risk of vocabulary drift.
- **Impact:** Future maintenance friction.
- **Fix:** Rename `.status-pill` ‚Üí `.status-badge` to differentiate.
- **Command:** `/normalize`

## Patterns & Systemic Issues

1. **Accessibility wiring lags behind aesthetic craft.** The design system is rich; the a11y plumbing (focus, ARIA states, captions, skip link) is thin. Likely because portfolio sites get iterated for visual impact, not for AT users.
2. **Image weight not addressed.** No lazy loading, no WebP, no `width`/`height` attributes, no font self-hosting. All standard portfolio-site oversights.
3. **Sidebar colors hardcoded instead of tokenized.** Internally consistent (deliberate brutalist juxtaposition) but breaks the otherwise-strong token discipline.
